## Question

Build a **Student Management REST API** using Spring Boot and MongoDB.

### Student Fields

| Field        | Type   | Notes          |
|--------------|--------|----------------|
| id           | String | Auto-generated by MongoDB |
| name         | String | Required       |
| rollNumber   | String | Unique         |
| department   | String | Required       |
| email        | String | Required       |

---

### What is provided (DO NOT MODIFY these files)

1. **Project structure** with all dependencies configured (`pom.xml`)
2. **Student model class** (`Student.java`) with `@Document` annotation — fully complete
3. **application.properties** configured for MongoDB connection
4. **Main application class** (`StudentCrudApplication.java`) — fully complete

### What you need to implement

#### Part A: Repository Layer (10 marks)

**File:** `src/main/java/com/exam/student/repository/StudentRepository.java`

- Extend `MongoRepository` with proper generic types
- Add a method to find students by department
- Add a method to search students by name (case-insensitive, partial match) using `@Query`

#### Part B: Service Layer (15 marks)

**File:** `src/main/java/com/exam/student/service/StudentService.java`

- Annotate the class as a Spring service
- Inject the repository
- Implement all CRUD operations:
  - Create a student
  - Get all students
  - Get a student by ID
  - Update a student by ID
  - Delete a student by ID
- Implement search by name
- Implement filter by department

#### Part C: REST Controller (20 marks)

**File:** `src/main/java/com/exam/student/controller/StudentController.java`

- Annotate as a REST controller with base path `/api/students`
- Inject the service layer
- Implement all 7 endpoints listed below with proper HTTP methods, status codes, and response handling

#### Part D: Testing with Postman (5 marks)

- Test all 7 endpoints using Postman or Thunder Client
- Insert the sample data provided below
- Demonstrate that search and filter endpoints work correctly
- Show the examiner your Postman results

---

### API Endpoints to Implement

| Method   | URL                                | Description              | Status Code  |
|----------|------------------------------------|--------------------------|--------------|
| `POST`   | `/api/students`                    | Create a new student     | 201 Created  |
| `GET`    | `/api/students`                    | Get all students         | 200 OK       |
| `GET`    | `/api/students/{id}`               | Get student by ID        | 200 OK       |
| `PUT`    | `/api/students/{id}`               | Update a student         | 200 OK       |
| `DELETE` | `/api/students/{id}`               | Delete a student         | 204 No Content |
| `GET`    | `/api/students/search?name=xxx`    | Search by name (partial) | 200 OK       |
| `GET`    | `/api/students/department/{dept}`  | Filter by department     | 200 OK       |
